# Stars Flutter 桌面 UI 风格指南

> 视觉基准：`Screenshot 2026-07-11 140120.png`
>
> 本规范用于 Stars 后续 Windows、Linux、macOS 桌面界面。目标是借鉴参考图的桌面任务工作台语言，不复制 ChatGPT/Codex 品牌元素；应用名称、图标和业务文案仍使用 Stars 自身体系。

## 1. 风格定义

Stars 桌面端采用“轻量任务工作台”风格：浅灰蓝应用外壳、宽项目侧栏、白色任务画布、右侧上下文检查器，以及贴近窗口底部的悬浮输入区。

整体感受应当是：

- 安静、精确、专业，适合长时间工作。
- 信息密度高于移动端，但通过留白和分区保持清晰。
- 以任务、项目、文件、运行状态为中心，不以 AI 形象为中心。
- 科技感来自细边框、图标、状态反馈和秩序，不来自渐变、发光或装饰插画。
- 桌面控件具备 hover、focus、selected、disabled 和快捷操作状态。

## 2. 参考图视觉特征

后续设计必须保留以下识别特征：

1. 左侧为约 `320-360px` 的完整项目与任务侧栏，不使用移动端抽屉或窄图标轨道。
2. 中央为占据主要空间的白色任务画布，内容采用文档流排版。
3. 右侧为约 `360-400px` 的浮层式上下文面板，可显示环境、文件变更和后台进程。
4. 顶部使用紧凑的窗口菜单和任务标题栏，不使用高大的移动端 AppBar。
5. 底部输入区悬浮在中央阅读区之上，白底、细边框、轻阴影。
6. 用户消息可显示为浅灰短气泡，Stars 输出保持无气泡的正文结构。
7. 主色只用于链接、焦点和少量状态；绝大多数界面使用中性色。

## 3. 桌面应用骨架

### 3.1 布局结构

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ Native title/menu bar                                      Window controls   │
├────────────────┬─────────────────────────────────────────────────────────────┤
│                │ Task header                                      Tools      │
│ Stars         ├──────────────────────────────────────┬──────────────────────┤
│ Global nav     │                                      │ Context inspector    │
│                │                                      │                      │
│ Projects       │ Main task/document stream            │ Environment          │
│ Tasks          │                                      │ Changes              │
│                │                                      │ Background processes │
│                │            Floating composer         │                      │
│ Account        │                                      │                      │
└────────────────┴──────────────────────────────────────┴──────────────────────┘
```

### 3.2 推荐尺寸

| 区域 | 推荐值 | 约束 |
| --- | --- | --- |
| 原生标题/菜单栏 | `40-44px` | 与系统窗口控件同层，紧凑 |
| 任务标题栏 | `56-60px` | 单行，底部 `1px` 分割线 |
| 左侧栏 | `320-360px` | 默认 `340px`，可在设置中收起 |
| 右侧检查器 | `360-400px` | 默认 `380px`，可显示/隐藏 |
| 中央最小宽度 | `640px` | 不足时优先隐藏右侧检查器 |
| 正文阅读宽度 | `880-960px` | 在中央区域内居中或稳定左对齐 |
| 输入区宽度 | `min(920px, 中央宽度 - 48px)` | 固定在中央阅读轴线上 |
| 面板间距 | `12-16px` | 仅右侧浮层需要外间距 |

### 3.3 响应式规则

- `>= 1500px`：左栏、中央区、右侧检查器同时显示。
- `1200-1499px`：左栏和中央区常驻，右侧检查器由按钮唤出，可覆盖在中央区右侧。
- `960-1199px`：左栏允许缩窄到 `280px`，右侧检查器默认关闭。
- `< 960px`：属于紧凑桌面模式，不将布局自动变成移动端；左栏改为可呼出的桌面侧栏。
- 中央正文不得随窗口无限拉宽。

## 4. 顶部区域

### 4.1 原生菜单栏

Windows/Linux 桌面端可使用接近参考图的轻量菜单栏：

- 左侧为侧栏开关、前进/后退和 `文件`、`编辑`、`视图`、`帮助`。
- 右侧保留系统最小化、最大化和关闭按钮。
- 背景使用应用壳层色，不额外制作一张白色卡片。
- 菜单文字 `13-14px`，弱化显示，hover 后增强对比度。
- macOS 遵循系统菜单栏习惯，不模拟 Windows 窗口按钮。

### 4.2 任务标题栏

任务标题栏位于中央区域顶部，结构为：

```text
[项目/文件图标] [当前任务标题，单行省略] [更多]          [运行环境] [面板开关]
```

规则：

- 高度 `56-60px`，白色背景，底部使用 `1px` 分割线。
- 标题 `14-15px / 600`，最多一行，过长时尾部省略。
- 工具按钮使用 `32-36px` 图标按钮，圆角 `8-10px`。
- 不放大标题，不显示页面级营销说明。

## 5. 左侧项目任务栏

左侧栏是稳定的工作导航，不是移动端抽屉。

### 5.1 信息结构

从上到下依次为：

1. Stars 品牌与搜索入口。
2. 全局功能：新建任务、已安排、插件、站点、拉取请求、聊天等。
3. 项目区：项目文件夹和项目内任务。
4. 普通任务区：不隶属于当前项目的任务。
5. 底部账号、连接状态和帮助入口。

### 5.2 视觉规则

- 背景使用浅灰蓝，与中央白色画布形成轻微区分。
- 左侧栏与中央区之间只用 `1px` 分割线，不使用阴影。
- 一级入口高度 `38-42px`，项目/任务项高度 `36-40px`。
- 行内图标 `18-20px`，使用统一线性图标，描边约 `1.5px`。
- 区域标题使用 `12-13px` 弱文字，不做大写或宽字距。
- 选中项使用浅灰实底和 `10-12px` 圆角，不铺品牌色。
- hover 仅轻微加深背景；pressed 再降低一个明度层级。
- 超长标题单行省略，右侧状态点、计数或进行中指示器保持固定位置。
- 底部账号栏固定，不随中间任务列表滚动。

### 5.3 项目与任务层级

- 项目使用文件夹图标，任务相对项目缩进 `24-28px`。
- 展开/折叠箭头只在 hover、focus 或存在子项时出现。
- 当前任务可在右侧显示小型进度环或状态点。
- 未读提示使用 `6-8px` 实心圆，颜色优先为蓝色，不使用发光效果。
- 不把每个任务做成独立卡片。

## 6. 中央任务工作区

中央区域是界面的视觉重心，采用接近代码编辑器和文档工具的白色连续画布。

### 6.1 内容流

- 内容区独立垂直滚动，顶部和底部为输入区预留安全空间。
- 正文最大宽度 `880-960px`，推荐 `920px`。
- 内容整体不要求严格居中，可在大屏上向左偏移，使阅读轴更接近项目侧栏。
- 正文 `14-15px`，行高 `1.55-1.7`。
- 标题、列表、代码、文件链接使用文档排版，不包入聊天气泡。
- 内容区可出现任务生成的图片或文件预览，但图片必须服务于任务结果，不作为背景装饰。

### 6.2 消息表现

用户消息：

- 右对齐，使用 `#F3F4F6` 浅灰背景。
- 圆角 `14-16px`，内边距 `10px 14px`。
- 最大宽度为正文区域的 `60%`，短消息按内容宽度收缩。
- 不使用头像、渐变和品牌色大气泡。

Stars 输出：

- 左对齐，无外层气泡、无重复头像。
- 直接使用 Markdown 文档流：段落、列表、链接、代码块和表格。
- 运行时长、步骤和次级状态使用弱文字与细分割线。
- 长内容以可读性为先，不刻意模拟即时通讯。

### 6.3 文件变更卡

参考图中的“已编辑文件”区域是任务状态组件，不是通用页面卡片。

- 白色或近白背景，`1px` 边框，圆角 `10-12px`。
- 高度按内容自适应，常规单文件状态约 `76-84px`。
- 左侧为文件/变更图标，中间为文件名及增删行数。
- 增加行数使用绿色，删除行数使用红色，避免大面积状态底色。
- 右侧放撤销、查看差异、审核等短命令。
- 多文件时使用列表展开，不套第二层卡片。

### 6.4 代码与命令

- 行内代码使用中性浅灰背景，圆角 `5-6px`，采用等宽字体。
- 代码块使用 `#F6F8FA` 或深色编辑器主题，但单页只选一种体系。
- 命令、文件路径和状态标记均允许复制。
- 文件链接使用克制蓝色，并配合文件类型图标。

## 7. 底部悬浮输入区

输入区是中央任务流的主要操作面，应接近参考图的宽幅桌面 composer。

### 7.1 结构

```text
┌────────────────────────────────────────────────────────────────┐
│ 输入后续要求                                                   │
│                                                                │
│ [+] [权限/上下文]                     [模型] [模式] [麦克风] [发送] │
└────────────────────────────────────────────────────────────────┘
```

### 7.2 样式

- 宽度上限 `920px`，最小高度 `112-124px`。
- 距中央区域底部 `16-20px`，并为系统安全区留空间。
- 背景为白色，边框 `1px`，圆角 `18-22px`。
- 使用单层柔和阴影，让输入区从正文上方浮起。
- 多行输入无独立内框，输入文字与底部工具共用一个完整容器。
- placeholder 使用弱文字，例如 `输入后续要求`，不使用拟人化文案。
- 左侧 `+` 为附件菜单；权限、模型、模式使用紧凑菜单或文本控件。
- 发送/停止使用深色圆形图标按钮，尺寸 `34-38px`，不写“发送”文字。
- focus 状态使用轻微加深边框，不使用蓝色发光外圈。

推荐阴影：

```dart
const BoxShadow(
  color: Color(0x140F172A),
  blurRadius: 24,
  offset: Offset(0, 8),
)
```

## 8. 右侧上下文检查器

右侧面板用于显示当前任务的环境和运行上下文，不作为固定的第三导航栏。

### 8.1 容器

- 距任务标题栏、窗口右侧和底部各 `14-16px`。
- 宽度默认 `380px`，白色背景，圆角 `20-24px`。
- 使用 `1px` 边框与轻阴影，形成浮层感。
- 面板内部独立滚动，滚动条窄且低对比度。
- 关闭后中央任务区获得空间，重新打开时保持原滚动位置。

### 8.2 内容分区

- 环境信息：文件变更、本地环境、分支、提交与推送、远程平台状态。
- 后台进程：正在运行或已运行的命令、持续时间和结果。
- 可选区域：上下文文件、端口、诊断和任务权限。

分区规则：

- 分区标题 `13px` 弱文字，右侧可放添加或更多按钮。
- 信息行高度 `38-42px`，图标 `17-19px`。
- 数值状态右对齐，增加为绿、删除/失败为红。
- 命令行文字单行省略，hover 显示完整 tooltip。
- 区域之间使用分割线，不将每行做成卡片。

### 9.2 圆角

| Token | 值 | 用途 |
| --- | --- | --- |
| `radiusSmall` | `6px` | 行内代码、小标签 |
| `radiusControl` | `8-10px` | 按钮、菜单项 |
| `radiusSelection` | `10-12px` | 左栏选中项、状态卡 |

普通页面区块不加圆角。圆角只用于可交互选中面、独立状态对象和真正浮起的工具面板。

### 9.3 字体

优先使用平台系统字体，中文保持清晰中性：

```dart
fontFamilyFallback: const [
  'Segoe UI',
  'Microsoft YaHei UI',
  'PingFang SC',
  'Noto Sans CJK SC',
  'sans-serif',
]
```

| 层级 | 字号 | 字重 | 行高 |
| --- | --- | --- | --- |
| 任务标题 | `15px` | `600` | `1.35` |
| 区域标题 | `13px` | `500` | `1.4` |
| 正文 | `14-15px` | `400` | `1.55-1.7` |
| 列表标题 | `14px` | `400-500` | `1.4` |
| 元信息 | `12-13px` | `400` | `1.4` |
| 代码 | `13-14px` | `400` | `1.55` |

不使用随窗口宽度缩放的字体，不使用负字距，不使用大号营销标题。

### 9.4 间距

基础间距使用 `4px` 网格：`4 / 8 / 12 / 16 / 20 / 24 / 32`。

- 左栏水平内边距：`12-16px`。
- 标题栏水平内边距：`18-20px`。
- 正文段落间距：`12-16px`。
- 内容大分区间距：`24-32px`。
- 图标与文字间距：`10-12px`。
- 紧凑工具按钮间距：`4-8px`。

## 10. 控件状态与交互

所有桌面控件必须具备以下状态：

- `default`：低对比、稳定。
- `hover`：背景或边框轻微变化，指针变为可点击状态。
- `pressed`：比 hover 再深一个层级。
- `focus`：`1-2px` 清晰焦点框，支持键盘导航。
- `selected`：使用稳定浅灰底，不依赖文字颜色单独表达。
- `disabled`：降低文字和图标透明度，但仍保持可识别。
- `loading`：在原控件尺寸内显示进度，不改变布局。

补充要求：

- 图标按钮必须提供 tooltip。
- 右键菜单用于项目、任务、文件和后台进程的次级操作。
- 行内更多按钮默认可弱化，hover/focus 时提高可见度。
- 列表、正文和检查器应分别保存滚动位置。
- 动画时长控制在 `120-200ms`，使用淡入、位移和尺寸过渡，不使用弹跳。

## 11. Flutter 落地约束

### 11.1 Theme 与 Token

继续以 `lib/utils/theme.dart` 作为桌面视觉入口，扩展现有 `DesktopThemeTokens` 和 `StarsDesktopTheme`。

建议至少统一以下字段：

```dart
class DesktopThemeTokens {
  static const double menuBarHeight = 42;
  static const double taskBarHeight = 58;
  static const double sidebarWidth = 340;
  static const double inspectorWidth = 380;
  static const double contentMaxWidth = 920;
  static const double composerMaxWidth = 920;
  static const double composerMinHeight = 116;
  static const double inspectorRadius = 22;
  static const double composerRadius = 20;
}
```

具体实现应遵循：

- 页面中不散落 `Color(0x...)`、圆角和阴影常量。
- 使用自定义桌面 shell，不使用 `BottomNavigationBar`、移动端 `Drawer` 或大号 `FloatingActionButton`。
- 左侧栏、中间内容、右侧检查器使用独立 `ScrollController`。
- 输入区应挂在中央工作区的 `Stack`/`Positioned` 或等价布局中，不覆盖最后一段正文。
- 右侧检查器显示/隐藏时使用 `AnimatedSize` 或宽度动画，并保持中央最小宽度。
- 固定宽度区域使用约束布局，避免内容、加载状态和 hover 导致跳动。
- 按钮优先使用项目现有图标库；同一界面不得混用多种图标风格。

### 11.2 组件边界

建议按以下组件拆分：

```text
StarsDesktopShell
├── DesktopMenuBar
├── ProjectTaskSidebar
│   ├── GlobalNavigation
│   ├── ProjectTree
│   ├── TaskList
│   └── AccountFooter
├── TaskWorkspace
│   ├── TaskHeader
│   ├── TaskStream
│   ├── FileChangeCard
│   └── FloatingComposer
└── ContextInspector
    ├── EnvironmentSection
    ├── ChangesSection
    └── BackgroundProcessList
```

不要在一个页面文件内同时实现所有栏目和状态逻辑。布局组件负责尺寸与响应式，业务组件负责内容与交互。

### 11.3 平台规则

- Windows/Linux 使用紧凑窗口菜单和明确 hover。
- macOS 使用系统标题栏与菜单习惯，内容区视觉保持一致。
- 触控设备不作为本桌面规范的主要目标。
- 移动端继续使用独立布局，不通过压缩桌面界面获得。

## 12. 文案规则

文案短、直接、面向任务：

推荐：

- `新建任务`
- `已安排`
- `环境信息`
- `后台进程`
- `提交或推送`
- `输入后续要求`
- `已编辑 3 个文件`
- `运行失败，查看日志`

避免：

- `让 AI 帮你开启灵感之旅`
- `你的智能伙伴正在思考`
- `释放无限创造力`
- `一键召唤超级助手`
- `魔法般完成任务`

状态描述使用客观动词：`正在运行`、`已完成`、`已取消`、`等待确认`、`执行失败`。

## 13. Do / Don't

### Do

- 使用宽项目任务栏、中央文档流和可收起检查器。
- 保持白色连续画布和浅灰蓝应用外壳。
- 用分割线、间距和排版建立层级。
- 让用户消息轻量，让 Stars 输出像工作记录和文档。
- 将文件变更、命令和运行状态设计成可操作的信息组件。
- 保持输入区悬浮、宽幅、低干扰。
- 支持鼠标、键盘、右键菜单和 tooltip。

### Don't

- 不使用窄移动端导航栏、底部导航或全屏抽屉。
- 不把每个区块包装成圆角卡片。
- 不使用大面积渐变、玻璃拟态、光斑、霓虹和发光描边。
- 不重复展示助手头像或机器人形象。
- 不使用左右对称的大气泡聊天布局。
- 不让品牌紫色成为大面积背景。
- 不使用夸张标题和解释 UI 功能的宣传文案。
- 不在窄窗口中粗暴压缩三栏导致文字和控件重叠。

## 14. 验收标准

桌面 UI 改造完成后必须检查：

1. 首屏结构与参考图一致：左侧项目任务栏、中央任务流、右侧上下文检查器、底部悬浮输入区。
2. 左侧栏能同时清晰呈现全局入口、项目、任务和账号信息。
3. 中央 Stars 输出为文档流，只有用户短消息使用浅灰气泡。
4. 右侧检查器可以显示/隐藏并独立滚动，不影响中央内容阅读宽度。
5. 输入区不遮挡最后一条内容，发送、停止、附件和模式操作完整可用。
6. hover、focus、selected、disabled、loading 状态均可辨识且不会引发布局跳动。
7. 页面没有移动端 AppBar、底部导航、超大触控控件和卡片堆叠感。
8. 页面没有大面积紫色、渐变、发光、拟人化助手和 AI 营销文案。
9. `1280x720`、`1440x900`、`1920x1080` 三种窗口尺寸下无重叠、溢出和不可达操作。
10. Windows 缩放比例 `100%`、`125%`、`150%` 下文字、图标和固定栏位保持清晰稳定。

## 15. 最终风格一句话

Stars 桌面端是一张安静、清晰、可持续工作的任务画布：左侧组织项目，中央推进任务，右侧掌握环境，底部随时继续操作。
